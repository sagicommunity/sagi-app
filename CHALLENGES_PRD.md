# PRD — Раздел «Challenges» (Челленджи)

**Продукт:** Sagi App — Community Module  
**Раздел:** Challenges Tab внутри страницы сообщества  
**Статус:** MVP / Design reference  
**Дата:** Май 2026

---

## 1. Цель раздела

Раздел Challenges мотивирует участников сообщества регулярно действовать — посещать мероприятия, тренироваться, участвовать в инициативах — и вознаграждает за это баллами (pts). Ключевое отличие от раздела Events: Challenges — это про **прогресс и геймификацию**, Events — про **логистику и присутствие**.

---

## 2. Роли

### 2.1 Администратор сообщества
Единственный, кто **создаёт и управляет** челленджами. Через административную панель сообщества Admin может:

- Создать новый Individual Challenge или Group Weekly Challenge
- Задать название, описание, тип, иконку, количество баллов, дедлайн, лимит мест
- Установить дату и время старта (используется для блокировки кнопки Unjoin за 1 час до начала)
- Деактивировать или удалить челлендж
- Видеть статистику: сколько участников, сколько выполнений

### 2.2 Участник сообщества (User)
Просматривает, фильтрует, вступает в челленджи, отмечает ежедневный прогресс, зарабатывает баллы.

---

## 3. Типы челленджей

### 3.1 Individual Challenge
Стандартная карточка — пользователь вступает и зарабатывает баллы.

**Поля (создаёт Admin):**
| Поле | Тип | Описание |
|------|-----|----------|
| `title` | string | Название |
| `description` | string | Краткое описание |
| `type` | enum | `sport`, `networking`, `charity`, `daily`, `streak`, `goal` |
| `icon` | emoji | Эмодзи-иконка |
| `points` | number | Баллы за выполнение |
| `slots` | `{filled, total}` | Лимит мест (опционально) |
| `deadline` | string | Отображаемая дата/дедлайн |
| `startISO` | ISO datetime | Дата и время старта (для блокировки Unjoin) |
| `isRecurring` | boolean | Повторяющийся (показывает 🔁) |

**Состояния карточки для пользователя:**

```
[Not joined]  →  Join → (кнопка зелёная)
      ↓
[Joined]      →  ✓ Joined  |  Unjoin (если до старта > 1 часа)
```

- При вступлении: счётчик `slots.filled` увеличивается на 1, прогресс-бар анимируется
- Баллы начисляются в момент вступления (verify: auto)
- **Unjoin:** доступна до тех пор, пока до `startISO` остаётся более 1 часа. При нажатии — баллы снимаются, счётчик уменьшается
- Если мест нет (`slots.filled >= slots.total`) — кнопка показывает «Full», вступление заблокировано

---

### 3.2 Group Weekly Challenge
Групповой трекер с еженедельной сеткой в формате «участник × день недели».

**Поля (создаёт Admin):**
| Поле | Тип | Описание |
|------|-----|----------|
| `title` | string | Название |
| `description` | string | Описание (если задано — показывает pre-join экран) |
| `type` | enum | Тот же набор типов, что у Individual |
| `icon` | emoji | Эмодзи-иконка |
| `tasks` | string[] | Список заданий на день (напр. «50 отжиманий», «5 км бег») |
| `pointsPerDay` | number | Баллы за каждый отмеченный день |
| `color` | hex | Акцентный цвет карточки |
| `participants` | array | Список участников с их прогрессом за текущую неделю |

**Состояния карточки:**

```
Если есть description:
  [Pre-join]  →  показывает описание + кнопку "Присоединиться →"
       ↓
  [Joined]    →  открывается еженедельная сетка

Если description нет:
  [Always expanded]  →  сетка видна сразу (пользователь уже участник)
```

**Еженедельная сетка:**
- Заголовок: `пн вт ср чт пт сб вс`
- Сегодняшний столбец подсвечивается акцентным цветом
- Строки: участники + строка «Вы» (всегда последняя, с зелёным фоном)
- Значения ячеек: `✅` (выполнено) | `❌` (пропущено) | `·` (нет данных)
- Кнопка **«Отметить сегодня — ПН/ВТ/...»** отмечает текущий день в строке «Вы»
- После отметки кнопка меняется на **«✓ Отмечено — ВТ»** (статичный badge)
- Баллы = количество отмеченных дней × `pointsPerDay`, показываются в шапке карточки (⚡ X pts)

**Сворачивание:**
- Тап по шапке карточки (заголовок + иконка) сворачивает/раскрывает карточку
- В свёрнутом состоянии: заголовок + «N участников · задания» + баллы пользователя + chevron

---

## 4. Лидерборд

- Показывается вверху вкладки Challenges
- Сворачивается/разворачивается
- Список: все участники сообщества, отсортированы по баллам (desc)
- Строка текущего пользователя («You») выделена зелёным фоном и всегда присутствует в списке
- По умолчанию показывает Top 5, кнопка «Show N more» раскрывает всех
- Баллы пользователя = сумма pts по всем выполненным Individual Challenges + pts за отмеченные дни в Group Challenges
- Клик по участнику → открывает карточку профиля участника

## 5. Фильтрация

Горизонтальный скролл чипов над карточками:

| Чип | Фильтрует |
|-----|-----------|
| All | Всё |
| ⚽ Sport | type = sport |
| 🤝 Networking | type = networking |
| 🤲 Charity | type = charity |
| ☀️ Daily | type = daily |
| 🔥 Streak | type = streak |
| 🎯 Goal | type = goal |

Фильтр применяется одновременно к Individual и Group challenges. Лидерборд остаётся видимым при любом фильтре.

## 6. Data Model (Backend)

```typescript
// Individual Challenge
interface Challenge {
  id: string;
  communityId: string;
  createdBy: string;         // adminId
  type: ChallengeType;
  icon: string;
  title: string;
  description: string;
  points: number;
  slots?: { total: number }; // filled считается отдельно
  startISO?: string;         // ISO datetime — для блокировки unjoin
  deadline?: string;         // display string
  isRecurring?: boolean;
  isActive: boolean;
  createdAt: string;
}

// Group Weekly Challenge
interface GroupChallenge {
  id: string;
  communityId: string;
  createdBy: string;         // adminId
  type: ChallengeType;
  icon: string;
  title: string;
  tasks: string[];
  description?: string;
  pointsPerDay: number;
  color: string;
  weekStartISO: string;      // начало текущей недели
  isActive: boolean;
  createdAt: string;
}

// Участие пользователя в Individual Challenge
interface ChallengeParticipation {
  userId: string;
  challengeId: string;
  joinedAt: string;
  isActive: boolean;         // false = unjoin
}

// Ежедневные отметки в Group Challenge
interface GroupCheckIn {
  userId: string;
  groupChallengeId: string;
  date: string;              // ISO date, e.g. "2026-05-18"
  status: 'done' | 'missed';
}
```

## 7. Ключевые UX-правила

1. **Никакого QR и геолокации** — весь verify-флоу убран. Join = выполнено (для Individual). Отметка кнопкой = выполнено (для Group).
2. **Unjoin lock** — за 1 час до `startISO` кнопка Unjoin исчезает. Это предотвращает отказы в последний момент.
3. **Счётчик реал-тайм** — при вступлении `filled` увеличивается мгновенно на клиенте, без ожидания сервера (optimistic update).
4. **Сетка только на текущую неделю** — Group Weekly Challenge всегда показывает Пн–Вс текущей недели. История недель — отдельная фича (v2).
5. **Самолёты убраны** — статус `away` не используется. Только `done`, `missed`, пустой.
6. **Баллы в шапке** — пользователь видит свои заработанные pts прямо в заголовке Group Challenge, без необходимости раскрывать карточку.

---

## 8. Out of Scope (v1)

- История предыдущих недель в Group Challenges
- Push-уведомления «Не забудь отметиться сегодня»
- Комментарии и реакции внутри челленджа
- Верификация через фото или геолокацию
- Индивидуальный прогресс-бар стрика
- Экспорт результатов
