# PRD — Раздел «Steps» (Шаги)

**Продукт:** Sagi App — Community Module  
**Раздел:** Steps Tab внутри страницы сообщества  
**Статус:** MVP  
**Дата:** Май 2026

---

## 1. Цель раздела

Steps — социальный шагомер внутри сообщества. Участники видят свой прогресс и сравнивают его с другими членами. Цель — создать лёгкую, ненавязчивую активность в сообществе: люди видят, что коллеги двигаются, и сами начинают двигаться больше.

Это не челлендж с призами — это ambient social accountability: «все видят, ты тоже стараешься».

---

## 2. Источник данных

Шаги синхронизируются с устройства пользователя:

| Источник | Платформа |
|----------|-----------|
| Apple Health (HealthKit) | iOS |
| Google Fit / Health Connect | Android |
| Ручной ввод | Fallback (опционально, v2) |

Синхронизация происходит:
- При открытии приложения — подтягивает актуальные данные
- По фоновому триггеру раз в час (если разрешения выданы)

Если пользователь не дал разрешение на доступ к шагам — вместо его данных в таблице показывается «нет данных».

---

## 3. Роли

### 3.1 Участник сообщества (User)
- Видит свои шаги за выбранный период
- Видит шаги других участников того же сообщества
- Переключает период (Сегодня / Вчера / Неделя / Месяц)

### 3.2 Администратор сообщества
- Включает или отключает вкладку Steps для своего сообщества
- Не управляет данными о шагах — они приходят с устройств участников

---

## 4. Интерфейс

### 4.1 Мой счётчик

Верхний блок на экране. Показывает шаги текущего пользователя за выбранный период.

```
шагов сегодня
✓  3 927  ›
157 🔥  ·  2,6 км  ·  4 ⬆
```

| Элемент | Описание |
|---------|----------|
| Галочка (✓) | Трекинг активен, данные синхронизированы |
| Число | Шаги за выбранный период, форматируются с пробелом-разделителем |
| Стрелка (›) | Переход на детальный экран шагов (история по часам, v2) |
| 🔥 калории | `Math.round(steps × 0.04)` ккал |
| км | `(steps × 0.00065).toFixed(1)` км |
| ⬆ этажи | Данные из HealthKit/Google Fit, fallback: `Math.round(steps / 950)` |

Если синхронизация ещё не проводилась — показывается заглушка «Подключите трекер» с кнопкой разрешения.

### 4.2 Фильтр периода

Горизонтальная строка из 4 кнопок на всю ширину:

| Кнопка | Данные |
|--------|--------|
| Сегодня | Шаги с 00:00 текущего дня |
| Вчера | Шаги за полный предыдущий день |
| Неделя | Сумма шагов Пн–Вс текущей недели |
| Месяц | Сумма шагов с 1-го числа текущего месяца |

При переключении периода пересчитываются и счётчик пользователя, и таблица сообщества.

### 4.3 Таблица сообщества

Список всех участников сообщества, отсортированных по шагам за выбранный период (по убыванию). Участники без данных за период — внизу.

| Элемент | Описание |
|---------|----------|
| 🥇 🥈 🥉 | Медали для топ-3 |
| Номер | Ранг для мест 4 и ниже |
| Аватар | Инициалы с фоном цвета участника |
| Имя | Имя участника; строка «Вы» — зелёная |
| Шаги | Число с разделителем; «нет данных» если нет синхронизации за период |

Строка текущего пользователя («Вы») всегда присутствует в списке и выделена зелёным фоном, даже если он на 10-м месте.

## 5. Логика данных

### 5.1 Что хранится на сервере

```typescript
interface StepRecord {
  userId: string;
  communityId: string;
  date: string;          // ISO date: "2026-05-18"
  steps: number;
  calories: number;
  distanceKm: number;
  floors: number;
  syncedAt: string;      // ISO datetime последней синхронизации
}
```

Один `StepRecord` — один день одного пользователя. Агрегация (неделя, месяц) считается на бэкенде по запросу.

### 5.2 API эндпоинты

```
GET /communities/:id/steps?period=today|yesterday|week|month
→ [ { userId, name, initials, color, steps, calories, distanceKm, floors } ]
Возвращает агрегированные данные всех участников за период, отсортированные по steps desc.

POST /steps/sync
body: { date, steps, calories, distanceKm, floors }
→ Принимает данные с устройства пользователя и сохраняет/обновляет StepRecord.
```

### 5.3 Обновление в реальном времени

- При открытии вкладки Steps — запрос к `/communities/:id/steps?period=today`
- WebSocket / SSE (v2): обновление таблицы без перезагрузки когда другой участник синхронизировался

## 6. Состояния участника в таблице

| Состояние | Когда | Отображение |
|-----------|-------|-------------|
| Есть данные | Синхронизация прошла за период | Число шагов + ранг |
| Нет данных | Нет синхронизации за период | «нет данных», без ранга, в конце списка |
| Не подключён | Пользователь не дал разрешение | «нет данных» |

---

## 7. Разрешения и приватность

- При первом открытии вкладки Steps — запрос разрешения на HealthKit / Health Connect
- Пользователь может в настройках профиля **скрыть свои шаги** — тогда строка «Вы» показывается только ему самому, остальным — «нет данных»
- Данные о шагах видны только участникам того же сообщества, не публично

---

## 8. UX-правила

1. **Число всегда с пробелом-разделителем** — `3 927`, не `3927`
2. **«Вы» всегда в списке** — даже если ноль шагов или нет данных, строка присутствует
3. **Период не сбрасывается** при переходе между вкладками внутри сессии
4. **Нет геймификации баллов** — Steps не начисляет Sagi Points (за это отвечает Challenges). Steps — отдельная, более лёгкая механика
5. **Нет цели по умолчанию** — нет прогресс-бара к 10 000 шагам в v1. Просто счётчик и сравнение
6. **Мобильные нативные данные** — шаги не вводятся вручную, только из HealthKit/Health Connect

---

## 9. Out of Scope (v1)

- Детальная история шагов по часам (тап на стрелку ›)
- Установка личной цели (например, 8 000 шагов/день)
- Push-уведомления «Ты на 2-м месте, обгони Kamila!»
- Еженедельные итоги в ленте новостей сообщества
- Ручной ввод шагов как fallback
- WebSocket real-time обновления таблицы
- Экспорт данных
