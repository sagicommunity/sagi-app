# PRD — Раздел «Events» (Мероприятия)

**Продукт:** Sagi App — Community Module  
**Раздел:** Events Tab внутри страницы сообщества  
**Статус:** MVP  
**Дата:** Май 2026

---

## 1. Цель раздела

Events — раздел логистики и присутствия внутри сообщества. Участники видят предстоящие мероприятия, записываются на них, следят за тем, кто из знакомых идёт, и общаются в рамках конкретного события. Ключевое отличие от Challenges: Events — про **встречи и посещаемость**, Challenges — про **прогресс и геймификацию**.

---

## 2. Роли

### 2.1 Администратор сообщества
Единственный, кто **создаёт и управляет** мероприятиями. Через административную панель может:

- Создать событие: заполнить название, описание, тип, дату/время, место, лимит мест, стоимость
- Деактивировать или удалить событие
- Видеть список участников и статистику посещаемости

### 2.2 Участник сообщества (User)
- Просматривает список событий
- Регистрируется (RSVP) и отменяет регистрацию
- Встаёт в лист ожидания при заполненности
- Видит, кто из участников сообщества идёт
- Устанавливает напоминание
- Делится событием
- Просматривает профиль других участников через чат события

## 4. Интерфейс

### 4.1 Список событий

Вертикальная лента карточек событий. По умолчанию показываются предстоящие события (dateISO >= сегодня), прошедшие скрыты за кнопкой «Показать прошедшие».

**Фильтрация:**
- Поиск по названию (текстовое поле вверху)
- Фильтр по дате: All / Today / This Week / This Month

### 4.2 Карточка события

```
[🍽️ Dining]  [20 000 ₸]              [🔔] [⤴]
Название события
📅 Сб, 17 мая · 19:30  [через 3 дн.]
📍 Chez Georges, Vertex Tower

Описание события — 2 строки с кнопкой «Читать далее»

████████░░  28/30 · 2 spots left

[AB] [FB] [KD] Aizat, Farida +1 going

[ I'm Going → ]  или  [ ✓ You're going  |  📅  |  💬 ]
```

| Элемент | Описание |
|---------|----------|
| Тип-бейдж | Цветной чип с эмодзи и названием типа |
| Цена | Стоимость в тенге или «Free» |
| Кнопка напоминания 🔔 | Включает/выключает напоминание. Toast-подтверждение. Активное состояние — цвет типа события |
| Кнопка шаринга ⤴ | Открывает шит с вариантами: WhatsApp, Telegram, Копировать |
| Обратный отсчёт | Бейдж рядом с датой: «Сегодня 🔥» / «Завтра» / «через N дн.» — только для предстоящих событий |
| Описание | 2 строки с кнопкой «Читать далее» / «Свернуть» |
| Прогресс-бар мест | Показывается только если у события задан лимит. Цвет: зелёный < 70%, оранжевый 70–90%, красный > 90% |
| Аватары друзей | До 4 кружков участников сообщества, которые идут. Тап → раскрывает список |

### 4.3 RSVP-кнопка

**До регистрации:**
```
[ I'm Going → ]
```
- При нажатии: кнопка меняется на состояние «Going», счётчик мест увеличивается на 1 мгновенно (optimistic update)
- Если мест нет (`slots.filled >= slots.total`): кнопка «Join Waitlist»

**После регистрации:**
```
[ ✓ You're going ]  [ 📅 Add to Calendar ]  [ 💬 Chat ]
```

- **Add to Calendar** — ссылка на Google Calendar с предзаполненными полями (название, место, дата)
- **Chat** — открывает шит участников данного события

**Отмена регистрации:** кнопка «Cancel RSVP» доступна до тех пор, пока до `startISO` остаётся более 1 часа. Счётчик уменьшается на 1.

### 4.4 Лист ожидания

Если `slots.filled >= slots.total`:
- Кнопка: «Join Waitlist»
- После нажатия: «⏳ On Waitlist · #N in line»
- Отображается количество ожидающих (`waitlistCount`)

### 4.5 Чат / участники события

Bottom sheet, открывается кнопкой 💬 (доступна только тем, кто зарегистрирован).

Содержит:
- Заголовок: название и дата события
- Список участников: «Вы» (зелёный, всегда вверху) + друзья из сообщества
- При тапе на участника → открывается карточка профиля
- Кнопка «Open WhatsApp Group» — ссылка на `wa.me` с текстом события

### 4.6 Шаринг события

Bottom sheet с тремя вариантами:

| Вариант | Действие |
|---------|----------|
| WhatsApp | Открывает `wa.me` с текстом: «Название — Дата · Место» |
| Telegram | Открывает `t.me/share/url` со ссылкой и описанием |
| Копировать | Копирует текст в буфер обмена, показывает toast «✓ Скопировано» |

### 4.7 Прошедшие события

Отображаются под кнопкой «Показать прошедшие (N)». В карточке прошедшего события:
- Нет RSVP-кнопки
- Нет счётчика мест
- Список посетивших (аватары + «attended»)
- Кнопка шаринга остаётся

### 4.8 Напоминание

- Кнопка 🔔 в правом верхнем углу карточки
- Активное состояние: иконка Bell + фон цвета типа события
- Неактивное: иконка BellOff + серый фон
- При переключении появляется toast: «🔔 Напоминание добавлено» / «Напоминание убрано»

## 6. Data Model (Backend)

```typescript
interface CommunityEvent {
  id: string;
  communityId: string;
  createdBy: string;           // adminId
  title: string;
  description: string;
  type: EventType;             // 'restaurant' | 'cafe' | 'education' | 'spa' | 'fitness'
  date: string;                // display string, e.g. "Сб, 17 мая · 19:30"
  dateISO: string;             // ISO date: "2026-05-17"
  startISO: string;            // ISO datetime для блокировки отмены: "2026-05-17T19:30:00"
  location: string;
  price?: number;              // в тенге; если не задано — бесплатно
  slots?: {
    total: number;
    filled: number;            // считается на бэкенде по participations
  };
  isActive: boolean;
  createdAt: string;
}

// Регистрация участника
interface EventParticipation {
  userId: string;
  eventId: string;
  status: 'going' | 'waitlist' | 'cancelled';
  joinedAt: string;
}

// Профиль участника для карточки
interface AttendeeProfile {
  userId: string;
  name: string;
  initials: string;
  color: string;
  role: string;
  location: string;
  tags: string[];
  connections: number;
  mutualCount: number;
  mutualNames: string[];
  aiSummary: [string, string, string];
}
```

## 7. API Эндпоинты

```
GET /communities/:id/events?period=upcoming|past|all
→ [ CommunityEvent ] — список событий сообщества

GET /communities/:id/events/:eventId/participants
→ [ { userId, name, initials, color, status } ] — участники события

POST /events/:eventId/rsvp
body: { action: 'join' | 'cancel' | 'waitlist' }
→ { status, slotsFilled }

GET /users/:id/profile
→ AttendeeProfile
```

## 8. UX-правила

1. **Optimistic update** — счётчик мест меняется мгновенно при RSVP, без ожидания ответа сервера
2. **Блокировка отмены** — кнопка Cancel RSVP скрывается за 1 час до `startISO`
3. **Обратный отсчёт** — показывается только для предстоящих событий; прошедшие без бейджа
4. **Прогресс-бар** — отображается только если у события задан лимит мест
5. **Чат только для зарегистрированных** — кнопка 💬 появляется лишь после RSVP
6. **Описание по умолчанию свёрнуто** — 2 строки, кнопка «Читать далее» раскрывает полный текст
7. **Toast-подтверждения** — любое действие (напоминание, шаринг, копирование) сопровождается уведомлением
8. **Нет QR, геолокации и фото** — подтверждение посещения не требуется. RSVP = присутствие

## 9. Out of Scope (v1)

- Push-уведомления по напоминаниям (сохраняется только локально)
- Комментарии и реакции внутри события
- Платёжный флоу для платных событий (цена отображается, оплата — вне приложения)
- Фильтр по типу события (Dining / Learning / Sport / Wellness)
- Повторяющиеся события (recurrence rules)
- Карта с точкой проведения внутри приложения
- История посещений участника
- Экспорт списка участников
